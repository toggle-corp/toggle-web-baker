## [Unreleased]

### Features

- Optional git credential for the clone (build) phase and `watchCommits`
  polling, lifting them out of GitHub's per-IP anonymous rate limit. Off unless
  configured (anonymous behaviour otherwise). An operator-global credential
  (`operator.gitAuth` in the chart тАФ via a pre-created `existingSecret` or
  chart-created `username`/`password`, mutually exclusive) is injected over
  HTTPS basic-auth only when the repo host matches the allowlist
  (`operator.gitAuth.hosts`, default `["github.com"]`), preventing credential
  forwarding to a user-controlled `spec.repo`. A per-App `spec.repoAuth.secretRef`
  fully overrides the global credential (host check skipped) for that App. The
  operator syncs owned, drift-corrected per-App copies of the global Secret and
  re-syncs on rotation; secret values are mounted as files (never URL-embedded)
  and never logged or rendered into the operator ConfigMap. Recommended: a
  dedicated bot account with a zero-scope classic PAT.

### BREAKING

- The CRD kind `FrontendApp` was renamed to `App` (group/version unchanged:
  `baker.toggle-corp.com/v1alpha1`). Plural is now `apps`, singular `app`,
  short name `bakerapp` (was `fapp`).
- The operator's Prometheus metrics were renamed from `frontendapp_*` to
  `baker_app_*` (e.g. `frontendapp_builds_total` тЖТ `baker_app_builds_total`,
  `frontendapp_phase` тЖТ `baker_app_phase`). Update any external dashboards or
  alert rules that reference the old names.
- This is a hard cutover: the old `FrontendApp` CRD and kind are removed with no
  back-compat shim (no live clusters yet). If a dev cluster still has old
  objects, recreate them as `App`s (strip server-managed metadata тАФ a raw
  `-o yaml | kubectl apply` fails on `resourceVersion` and would also carry the
  old finalizer):

  ```sh
  kubectl get frontendapps -A -o json | jq '.items[] | {apiVersion, kind: "App",
    metadata: {name: .metadata.name, namespace: .metadata.namespace,
               labels: .metadata.labels}, spec}' | kubectl apply -f -
  ```

  Then, per old object: remove its `baker.toggle-corp.com/finalizer` (no
  controller watches the old kind anymore, so deletion hangs on it otherwise)
  and delete its children (`<name>-nginx` etc. тАФ their label selectors are
  immutable and still say `app.kubernetes.io/name=frontendapp`; the new
  operator recreates them). Finally delete the old CRD:

  ```sh
  kubectl get frontendapps -A -o name | while read -r r; do
    kubectl patch "$r" --type merge -p '{"metadata":{"finalizers":null}}'; done
  kubectl delete crd frontendapps.baker.toggle-corp.com
  ```
